# 了解暂定/确认日期 {#understanding-tentative-confirmed-dates}

智能营销活动和电子邮件程序具有强大的功能，可标记为 **暂定** 或 **已确认**. 以下是它们的工作方式。

## 暂定 {#tentative}

暂定日期表明意图。 请将此视为 _penciling_ 日历里有什么东西。 暂定条目不会运行 — 它们只是占位符。

![](assets/image2014-9-23-15-3a22-3a23.png)

>[!NOTE]
>
>只有批量智能营销活动和电子邮件程序可以是临时性的。

## 确认条目 {#confirming-entries}

这就像批准资产一样，因此在确认条目之前，需要完全设置条目。 一旦所有的鸭子都排成一行，您就可以向右滑动试用的标签来确认登入点。

![](assets/image2014-9-23-15-3a23-3a2.png)

![](assets/image2014-9-23-15-3a23-3a8.png)

![](assets/image2014-9-23-15-3a23-3a12.png)

>[!NOTE]
>
>为什么是狗？ 他是猎人。 他在提取你的数据。

## 已确认 {#confirmed}

已确认条目肯定会运行。 它们具有规则、批准的资源和确认的日期和时间。

![](assets/image2014-9-23-15-3a23-3a30.png)

## 已完成  {#finished}

已完成条目已运行。 它们只能是过去（显然）。 条目运行且为 **已完成**，则不能移动它或将其设为试用。 （不会改变现实和破坏时间流连续体。 ）

![](assets/image2014-9-23-15-3a25-3a53.png)

这些状态是强大的工具。 当您克隆项目时，所有智能营销活动和电子邮件项目日期都是暂定日期。 它们都可以从计划视图中确认。 很酷吧？
