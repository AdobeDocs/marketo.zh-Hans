# 智能列表过滤器运算符一览表 {#smart-list-filter-operators-glossary}

运算符是智能列表的一部分，可帮助您确定具体的内容。 它允许您以简单的语言描述过滤器或触发器。 每种类型的字段的可用运算符各不相同。

下面是一个术语表，用于描述每组运算符。

## 日期字段 {#date-fields}

![](assets/image2014-9-10-17-3a15-3a47.png)

选择运算符时，右侧将动态更改。

| 运算符 | 右侧 | 描述 |
|---|---|---|
| is | 单个日期 | 精确日期匹配 |
| 不是 | 单个日期 | 任何日期（指定的日期除外） |
| 介于 | 两个日期字段 | 包括和两个指定日期之间的任何日期 |
| 过去 | 自然语言输入&#42; | 请参阅下图 |
| 以前 | 自然语言输入&#42; | 请参阅下图 |
| 未来 | 自然语言输入&#42; | 请参阅下图 |
| 以后 | 自然语言输入&#42; | 请参阅下图 |
| 时间范围 | 预设（上一季度、昨天等） | 在挑库列表中定义 |
| after | 单个日期 | 指定日期之后的所有记录 |
| 之前 | 单个日期 | 指定记录之前的所有记录 |
| 在或之后 | 单个日期 | 与“after”相同，但包含 |
| 在或之前 | 单个日期 | 与“之前”相同，但包含 |
| 为空 | 无 | 没有日期的所有记录 |
| 不为空 | 无 | 具有任何日期的所有记录 |

&#42; 自然语言输入很酷。 您可以输入以下一些模式：

* 1小时
* 82天
* 3周
* 14个月
* 1年

只要把号码和单位一起输入，就行了！

>[!NOTE]
>
>&quot;过去&quot; **does** 包括您创建智能列表的日期（直到时间，而不是之后）。

>[!CAUTION]
>
>当您使用日期字段过滤器（例如，出生日期、SFDC创建日期）创建智能列表时，请使用 **之前** 或 **在或之前** 限制，智能列表还将包括在所述日期字段中没有值的人员。

请使用下图了解日期运算符之间的差异。

![](assets/image2014-9-10-17-3a15-3a58.png)

>[!NOTE]
>
>**示例**
>
>当您处理过去和将来的事件时，日期字段可能会变得棘手。 以下是几个示例。
>
>**以前**
>
>对于您的新促销活动，使用此操作员仅向一年内未订阅或续订您的服务或从未订阅过服务的用户发送电子邮件。
>
>**以后**
>
>如果您希望看到90天内有待续订的客户。 您应使用两个不同的过滤器。 首先使用“在90天后”，然后使用“在将来91天内”。 那会捕捉谁在90天后有日期。

## 字符串字段 {#string-fields}

![](assets/image2014-9-10-17-3a16-3a6.png)

| 运算符 | 描述 |
|---|---|
| is | 完全匹配（不区分大小写） |
| 不是 | 除外的任何内容都完全匹配 |
| 开头 | 字符串匹配的前字母 |
| 开头 | 字符串的前字母不匹配 |
| 包含 | 字符串中的任意字母匹配(示例：加利福尼亚，福特，为此) |
| 不包含 | 字符串中没有字母匹配。 （“包含”的反向） |
| 为空 | 没有值(NULL)的记录 |
| 不为空 | 具有ANY值的记录 |

>[!TIP]
>
>使用正运算符而不是负运算符。 “不是”过滤器必须搜索实例中的整个数据集，这可能非常耗时。 正“is”过滤器可以利用更有效的搜索算法。
