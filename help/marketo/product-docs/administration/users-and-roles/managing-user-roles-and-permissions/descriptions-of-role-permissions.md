# 角色权限描述 {#descriptions-of-role-permissions}

以下是可分配给角色的所有可用权限列表。 权限通常与Marketo中的特定功能区域关联，可以帮助您控制不同用户有权访问的区域和功能。

有关权限的其他信息包括：

* “访问”权限授予了查看和有时编辑应用程序该部分的角色权限。
* 要使角色具有子权限（“创建”、“删除”等），该角色必须具有应用程序该部分的“访问”权限。 例如，如果您想要为某人授予“编辑营销活动”的权限，则他们必须拥有访问营销活动的整体权限。
* 您可能会看到您没有使用权限的操作或资产。 但是，如果您尝试访问这些权限，则会看到一条消息，警告您您的有限访问权限。

## 可用权限 {#available-permissions}

当您 [创建或编辑角色](/help/marketo/product-docs/administration/users-and-roles/managing-user-roles-and-permissions.md)，您可以通过选中相应的复选框来选择以下哪些权限允许该角色。

## 访问管理员  {#access-admin}

查看并更改“管理员”的“我的帐户”部分中的设置。

* 访问审核跟踪 — 允许用户同时访问资产审核跟踪和管理员审核跟踪
* 访问渠道 — 授予用户仅修改渠道标记的访问权限，而无法修改其他自定义标记
* 访问通信限制 — 授予用户在“管理员”中启用通信限制的访问权限
* 访问CRM — 在“管理员”中为用户授予对CRM(如Salesforce或Microsoft Dynamics)的访问权限
* 访问 [Data.com](https://Data.com)  — 授予用户访问Data.com流程操作的权限
* 访问电子邮件管理员 — 为用户授予电子邮件管理员权限，以更改默认设置，如取消订阅和品牌化域名
* 访问事件合作伙伴 — 授予用户在管理员中访问LaunchPoint的权限
* 访问字段管理 — 授予用户访问“管理员”中的字段管理的权限
* 访问文件上传 — 允许用户将图像和文件上传到Design Studio
* 访问登陆页面 — 允许用户在“管理员”中访问登陆页面
* 访问位置 — 允许用户访问“管理员”中的位置，以设置默认语言、区域设置、时区和货币
* 访问登录历史记录 — 授予用户访问审核记录中用户登录历史记录的权限
* 访问登录设置 — 为用户授予对“管理员”中“安全性”、“IP限制”和“智能列表报表”设置的登录设置的访问权限
* 访问Marketo自定义活动 — 允许用户在“管理员”中访问Marketo自定义活动
* 访问Marketo自定义对象 — 允许用户在管理中访问Marketo自定义对象
* 访问Munchkin - GI用户在“管理员”中访问Munchkin，以设置跟踪代码、人员跟踪和启用API配置
* 访问收入周期分析 — 允许用户在“管理员”中访问收入周期分析，以设置同步摘要和归因
* 访问角色 — 授予用户管理和编辑角色的权限，但不授予用户权限
* 访问销售分析 — 授予用户在“管理员”中管理销售分析的权限，以便设置状态、API配置、人员评分和其他设置
* 访问单点登录 — 授予用户在“管理员”中管理单点登录的权限，以启用SAML并使用SAML设置和重定向页面URL
* 访问智能营销活动 — 在管理员中为用户授予对智能营销活动的访问权限，以限制对合格人员的限制
* 访问SOAP API — 允许用户在“管理员”的Web服务中管理SOAP API
* 访问标记 — 让用户可以访问除渠道标记之外的所有自定义标记
* 访问Treasure Chest — 让用户能够访问Admin中Treasure Chest中的实验功能
* 访问用户 — 授予用户在“管理员”中编辑和管理用户（但不是角色）的权限
* 访问Webhook — 将用户授予管理员中的Webhook，以设置详细信息和响应映射
* 访问工作区和分区 — 允许用户在“管理员”中创建、编辑和删除工作区和分区
