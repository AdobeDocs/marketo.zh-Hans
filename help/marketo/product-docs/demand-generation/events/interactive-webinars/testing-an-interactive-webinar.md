# 测试交互式网络研讨会 {#testing-an-interactive-webinar}

交互式网络研讨会涉及将Adobe Connect用作网络研讨会提供工具。 最佳实践是在托管您的第一个实时网络研讨会之前创建测试网络研讨会。

任何交互式网络研讨会都可以用作测试网络研讨会。 测试可以通过添加不超过10位主持人和演示者（参与者）来执行。

创建测试网络研讨会的工作流程与创建任何其他网络研讨会](/help/marketo/product-docs/demand-generation/events/interactive-webinars/create-an-interactive-webinar.md)的交互式网络研讨会事件程序相同。 [建议将选定的交货日期设在遥远的将来。

>[!CAUTION]
>
>如果您将测试网络研讨会的日期和时间设置得太近，并且该日期/时间已过，则交互式网络研讨会会考虑已交付的网络研讨会，并且会从您的许可证中扣除一个点数。

在此停止

要避免这种情况，请在不久的将来创建交互式网络研讨会，并且只阻止1个点数。 您可以通过将计划移动到所需日期来重新安排计划以将其用于实际网络研讨会，或者只需删除网络研讨会计划即可使用此积分。

在创建和设计网络研讨会后，主持人、共同主持人和/或演示者可以加入网络研讨会以排练投放。 用户还可以将外部受邀者作为联合主机添加到网络研讨会中，以便能够参加测试。 测试可以通过与网络研讨会的发布方式相同的方式执行，这样评估人员就可以大致了解网络研讨会将如何发布给实际受众。 实时网络研讨会中的所有功能也可在测试网络研讨会中获取。 此操作可一直执行到网络研讨会开始前15分钟。 建议主机重新安排或删除网络研讨会，以免因许可证而失去交互式网络研讨会积分。

由于这是测试网络研讨会，因此在实时网络研讨会后有几个内容不可用于测试。 这包括录制网络研讨会，以参与仪表板或仪表板选项卡中的汇总仪表板的形式获取网络研讨会的参与见解。 交互式网络研讨会的按需网络研讨会跟踪功能也不再可用，该功能可用于录制的网络研讨会。 由于在测试网络研讨会期间没有跟踪参与数据，因此不会从Adobe Connect向Marketo返回任何数据，并且在网络研讨会后无法执行任何培育操作。

让测试网络研讨会可用的过程背后的意图是，营销人员可以测试从创建到交付的工作流，然后使用其中一个学分交付实时网络研讨会，以探索上面列出的其他功能。
