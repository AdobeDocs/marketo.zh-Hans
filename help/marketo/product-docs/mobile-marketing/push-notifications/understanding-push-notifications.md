# 了解推送通知 {#understanding-push-notifications}

>[!NOTE]
>
>应用程序内消息传递是一个附加应用程序。 请与您的Marketo客户经理确认，以确保该帐户已激活。

通过Marketo Mobile Engagement，您可以创建、配置和发送通知，操作方式与创建电子邮件类似。  在从移动设备应用程序创建和发送推送通知之前，您需要Marketo管理员和移动设备应用程序开发人员执行一些设置。

>[!CAUTION]
>
>推送通知是一个加载项，在开始之前必须由Marketo管理员打开。

## 步骤1:管理员和开发人员执行初始设置 {#step-admin-and-developer-perform-initial-setups}

Marketo管理员和移动设备应用程序开发人员将共同为您进行设置。 请参阅 [在创建推送通知和应用程序内消息之前](/help/marketo/product-docs/mobile-marketing/admin/before-you-create-push-notifications-and-in-app-messages.md) 以了解详细信息。

## 步骤2:创建推送通知 {#step-create-a-push-notification}

[编写邮件](/help/marketo/product-docs/mobile-marketing/push-notifications/create-a-push-notification.md) 并预览在Android和iOS设备上的显示方式。

## 步骤3:发送！ {#step-send}

[可以发送推送通知](/help/marketo/product-docs/mobile-marketing/push-notifications/send-a-mobile-push-notification.md) 使用触发器和批量智能营销活动。 很酷吧？

![](assets/image2015-4-27-8-3a41-3a43.png)

>[!NOTE]
>
>* 在应用程序至少打开一次之前，屏幕上不会弹出推送通知。
>* 对于iOS应用程序，如果指定接收推送消息的应用程序处于打开状态并且处于活动状态，则推送通知不会在屏幕上弹出。 相反，消息将显示在应用程序的本地通知区域。
