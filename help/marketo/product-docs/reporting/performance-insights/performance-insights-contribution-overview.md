# 性能分析贡献概述 {#performance-insights-contribution-overview}

在Marketo性能分析中，默认显示“贡献”视图。

![](assets/one-1.png)

选择要按以下方式查看性能的量度。 在此示例中，我们将在收入仪表板中查看通过多点接触赢得的机会。

![](assets/2.png)

>[!NOTE]
>
>了解 [首次联系和多次联系](/help/marketo/product-docs/reporting/revenue-cycle-analytics/revenue-tools/attribution/understanding-attribution.md).

选择要查看量度的时段。 在本例中，我们查看的是当前年份（年初至今）。

![](assets/3-1.png)

>[!NOTE]
>
>我们暂时删除了“上一年”选项。 您仍可以选择使用“自定义范围”选项查看上一年的整个性能数据。

量度通过以下两个图表显示：甜甜圈和棒子。

![](assets/four.png)

圆环图显示所选量度的前十个渠道。

![](assets/5-1.png)

条形图显示您所选量度的所有渠道（一次十个项目）中的程序性能。 要了解更多信息，请单击右侧的箭头滚动到下一个组。

![](assets/six.png)

>[!TIP]
>
>如果希望图形中的条在您浏览组时向上缩放，请选择 **缩放Y轴以适合** 复选框。

将鼠标悬停在条形上可查看其他详细信息。

![](assets/seven.png)

在圆环图中选择一个或多个渠道，与这些渠道关联的所有程序都会显示在右侧的条形图中。 再次单击渠道以取消选择。

![](assets/eight.png)

下面的数据网格的功能与电子表格类似，它显示了所选归因模型（首次联系/多联系）下的所有可用量度。 包含您选择的量度的列会高亮显示。

![](assets/9.png)

| **赢得的机会** | 为影响赢得的机会而接收的程序的信用部分（以数字值表示） |
|---|---|
| **收入赢得** | 为影响赢得的机会而收到的计划的信用部分（货币价值） |
| **成本** | 方案总费用 |
| **每个机会的成本** | 为影响新机会的创造而收到的方案的成本与信用部分（以数字值表示）的比率 |
| **收入胜出成本比** | 为影响赢得的机会而收到的方案的信用部分（货币价值）与方案成本的比率 |

扩展渠道以查看其前十个项目，其余项目合并在一起。

![](assets/10.png)

>[!NOTE]
>
>单击渠道激活/停用旁边的复选框，将其在上面的圆环图中激活/停用。
