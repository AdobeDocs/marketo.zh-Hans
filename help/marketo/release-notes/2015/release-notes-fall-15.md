# 发行说明：2015年秋季 {#release-notes-fall}

2015年秋季版包含以下功能。 请查看您的Marketo Edition以了解功能可用性。

## 订阅智能列表 {#subscribe-to-a-smart-list}

[订阅智能列表](../../product-docs/reporting/basic-reporting/report-subscriptions/subscribe-to-a-smart-list.md)

订阅智能列表允许营销人员导出智能列表，并将其通过电子邮件发送给未使用营销工具的利益相关方，例如销售或电话营销团队。

导出可以按每日、每周或每月计划，可以具有结束投放日期，并可以自定义以共享有限数量的列。

![](assets/image2015-10-1-17-3a12-3a50.png)

可以在智能订阅上创建多个列表。 每个订阅、不同工作区、每个Marketo实例均限制100个订阅，每个有100K个潜在客户。

![](assets/image2015-10-1-17-3a11-3a50.png)

[Marketo自定义对象](http://docs.marketo.com/display/docs/marketo+custom+objects)

从管理员UI轻松创建自定义对象。 我们目前支持在Marketo中创建1:N自定义对象并将其连接到潜在客户或公司。

>[!NOTE]
>
>Marketo自定义对象不适用于Spark。

## Google Chrome的Marketo洞察 {#marketo-insights-for-google-chrome}

[Google Chrome的Marketo洞察](../../product-docs/marketo-sales-insight/msi-chrome-plugin/using-marketo-insights-for-google-chrome.md)

我们很高兴地宣布发布Google邮件销售分析扩展的更新！ 在Chrome商店 [视图它](https://nation.marketo.com/external-link.jspa?url=https://chrome.google.com/webstore/detail/marketo-insights-for-goog/jjkfbhajlmoeegbjgjipliamplidmbjb)。

此更新包括许多新增功能和特性：

* 在进行互动之前，销售人员可以直接在Google Mail中查看有关潜在客户的相关信息，包括职位、twitter用户档案、公司信息、照片等。
* 销售人员可以实时查看潜在客户在各渠道中参与的内容，如打开或点击的电子邮件、在线或亲临事件出席、访问的网页、下载的eBook等。
* 通过Google Mail发送的电子邮件将记录在Marketo中并实时跟踪。 这样，销售人员便可以了解潜在客户查看电子邮件的时间，以便在适当的时间跟进。 Marketo Sales Insight for Google Mail还使销售人员能够轻松利用由营销创建的模板来发送美观的邀请、优惠和其他类型的内容。

![](assets/image2015-10-2-14-3a47-3a53.png)

## RTP —— 内容分析和Recommendations {#rtp-content-analytics-and-recommendations}

[内容分析](../../product-docs/web-personalization/understanding-web-personalization/understanding-content-analytics.md) 和Recommendations

RTP内容分析显示Web内容资产在定期Web访问以及RTP内容推荐引擎生成的访问中的性能。

* 了解哪些内容表现最佳并引入最多潜在客户
* 通过在RTP的预测内容引擎中启用内容自动向正确的访客推荐最佳内容，从而提高内容消费
* 深入分析每个内容资产，以查看更多深入指标、图表和性能

RTP的“资产”页面现在分为内容分析和内容Recommendations。

* **内容分析：** 显示所有已发现和定义的Web内容的视图和直接潜在客户，帮助您分析效果最佳的内容
* **内容Recommendations**:显示来自RTP推荐内容和相关潜在客户归因的展示次数和点击次数。 您还可以编辑和启用此页面中针对栏和富媒体 [推荐](https://docs.marketo.com/display/DOCS/Enabling+the+Content+Recommendation+Engine) 的 [内容推荐](https://docs.marketo.com/display/DOCS/Enabling+the+Rich+Media+Recommendation+Engine) 。

* 自年度开始（2015年1月1日）起，这两个页面中的所有直接潜在客户数据已追溯更新。

## 富文本编辑器改进 {#rich-text-editor-improvements}

我们对富文本编辑器进行了多项改进。 在7月发布更新的编辑后，我们收到了大量反馈，并能够将这些更改用于此升级。 未来几个月还有更多工作要做。 以下是第4季度新增功能的列表:

* HTML代码中现在支持VML:

   ```
   <v:background xmlns:v="urn:schemas-microsoft-com:vml" fill="t">
   ```

   ```
   <v:fill type="tile" src="<a href="http://i.imgur.com/YJOX1PC.png" rel="nofollow">http://i.imgur.com/YJOX1PC.png</a>" color="#7bceeb"/>
   ```

   ```
   </v:background>
   ```

* 现在，任何内容都可以插入有效的HTML注释中（以前已删除下面所示的某些语法）:

   ```
   <!--[if gte mso 9]> 
   ```

   ```
   <![endif]-->
   ```

* 不要将空表单元格 `**`

* 添加到HTML源代码编辑器的最大化／最小化按钮
* 现在，已识别并显示预存的表属性在表属性对话框中
* 现在，两行按钮都默认显示。
* 编辑器现在将接受任何元素（甚至已弃用或非标准元素）:

   ```
   <myCustomElement>Hello World!</myCustomElement>
   ```

* 编辑器现在将接受任何属性（甚至已弃用或非标准属性）:

   ```
   <myCustomElement myCustomAttribute="foo">Hello World!</myCustomElement>
   ```

   ```
   <td background="someImage.png"> 
   ```
