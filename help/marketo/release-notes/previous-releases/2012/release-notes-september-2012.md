# 发行说明：2012年9月 {#release-notes-september}

此版本包括备受期待的集成式社交功能和潜在客户管理实用组件！ 注意：社交功能可作为加载项或作为选定包的一部分提供。

## 通过社交共享发布YouTube视频 {#publish-a-youtube-video-with-social-sharing}

通过在您的登陆页面上使用新的“视频共享”鼓励访客在社交上分享，从而扩大视频的受众。

![](assets/image2014-9-23-10-3a39-3a21.png)

## 添加共享按钮 {#add-a-share-button}

完全自定义共享消息和一组新的社交共享按钮的外观。 此外，当您的潜在客户共享您的内容时，会捕获社交用户档案数据。

![](assets/image2014-9-23-10-3a39-3a46.png)

## 社交登录 {#social-sign-on}

通过允许用户通过社交网络预填表单中的信息来获取洞察并减少摩擦。

![](assets/image2014-9-23-10-3a40-3a2.png)

## 将登陆页面发布到Facebook {#publish-landing-pages-to-facebook}

通过将登陆页面直接发布到Facebook，以及社交应用程序、表单和Marketo登陆页面的完整功能，扩展登陆页面的访问范围。

## ReadyTalk事件适配器 {#readytalk-event-adapter}

将Marketo事件无缝连接到ReadyTalk会议。 使用Marketo表单捕获注册者，并在ReadyTalk中自动注册。 双向同步允许考勤信息填充到Marketo中。

![](assets/image2014-9-23-10-3a40-3a16.png)

## Microsoft Dynamics内部部署 {#microsoft-dynamics-on-premise}

我们现在通过面向Internet的部署，在内部部署支持Microsoft Dynamics 2011。

## 网钩（宝箱） {#webhooks-treasure-chest}

Webhook是用户定义的HTTP回调。 这是将数据从Marketo推送到任何其他服务的绝佳方法。 此功能目前在“财宝箱”中可用，目前仅在触发器促销活动中受支持。

有关如何使用Webhook的示例包括：将用户名和密码信息发布到其他系统以创建试用帐户；在您获得新潜在客户时发送短信文本消息。

## getMultipleLeads API的更新 {#update-to-getmultipleleads-api}

我们为getMultipleLeads API调用添加了新的筛选条件。 除了按日期过滤之外，我们现在还支持其他标准：

* 日期范围
* 静态列表名称
* 潜在客户键数组
