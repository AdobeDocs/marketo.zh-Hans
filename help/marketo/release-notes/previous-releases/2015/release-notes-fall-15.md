# 发行说明： 2015年秋季 {#release-notes-fall}

2015年秋季版本中包含以下功能。 请检查您的Marketo版本是否提供了相关功能。

## 订阅智能列表 {#subscribe-to-a-smart-list}

[订阅智能列表](/help/marketo/product-docs/reporting/basic-reporting/report-subscriptions/subscribe-to-a-smart-list.md)

通过订阅智能列表，营销人员可以导出智能列表，并通过电子邮件将其发送给未使用Marketo的利益相关者，例如销售或电话营销团队。

可以安排每日、每周或每月导出，可以有结束交付日期，并且可以自定义以共享有限数量的列。

![](assets/image2015-10-1-17-3a12-3a50.png)

可以在智能列表上创建多个订阅。 根据Marketo实例的限制，每个订阅、跨工作区有100个潜在客户和100,000个订阅。

![](assets/image2015-10-1-17-3a11-3a50.png)

[Marketo自定义对象](/help/marketo/product-docs/administration/marketo-custom-objects/understanding-marketo-custom-objects.md)

## Google Chrome的Marketo Insights {#marketo-insights-for-google-chrome}

[Google Chrome的Marketo Insights](/help/marketo/product-docs/marketo-sales-insight/msi-chrome-plugin/using-marketo-insights-for-google-chrome.md)

我们很高兴地宣布推出Google Mail Sales Insight扩展的更新版本！ 在中查看  [Chrome Store](https://chrome.google.com/webstore/detail/marketo-insights-for-goog/jjkfbhajlmoeegbjgjipliamplidmbjb).

此更新包括许多新特性和功能：

* 在接洽之前，销售人员可以直接在Google Mail中查看有关其潜在客户的相关信息，包括职称、twitter用户档案、公司信息、照片等。
* 销售人员可以实时查看潜在客户跨渠道接触的内容，例如打开或点击的电子邮件、参加的在线或面对面活动、访问的网页、下载的电子书等等。
* 通过Google Mail发送的电子邮件将记录在Marketo中并实时跟踪。 这可以让销售人员查看潜在客户何时查看其电子邮件，以便他们可以在适当的时间跟进。 通过Marketo Sales Insight for Google Mail ，销售人员还可以轻松地利用营销团队创建的模板来发送精美的邀请、优惠和其他类型的内容。

## RTP — 内容分析和Recommendations {#rtp-content-analytics-and-recommendations}

[内容分析](/help/marketo/product-docs/web-personalization/understanding-web-personalization/understanding-content-analytics.md) 和Recommendations

RTP内容分析通过定期Web访问以及通过RTP内容推荐引擎生成的访问来显示Web内容资产的性能。

* 查看哪些内容表现最佳并吸引最多的潜在客户
* 通过在RTP的预测性内容引擎中启用内容来自动向合适的访客推荐最佳内容，从而增加内容使用量
* 深入查看每个内容资产以了解更深入的量度、图表和性能

RTP的Assets页面现在分为内容分析和内容Recommendations。

* **内容分析：** 显示所有已发现和定义的Web内容的视图和直接潜在客户，帮助您分析表现最佳的内容
* **内容Recommendations**：显示RTP推荐内容的展示次数和点击次数以及关联的潜在客户归因。 您还可以在此页面中为编辑和启用内容推荐 [条形图](/help/marketo/product-docs/predictive-content/enabling-predictive-content/enable-the-content-recommendation-bar.md) 和 [富媒体](/help/marketo/product-docs/predictive-content/enabling-predictive-content/enable-predictive-content-for-web-rich-media.md) recommendations.

* 自年初以来（2015年1月1日），这两页中的所有直接销售线索数据均已追溯更新。

## 富文本编辑器改进 {#rich-text-editor-improvements}

我们正在对富文本编辑器进行几项改进。 在7月发布更新编辑器后，我们收到了很好的反馈，并且可以将这些更改用于此升级。 接下来的几个月里，还有更多事情要做。 下面是第4季度新增功能列表：

* 您的HTML代码现在支持VML：

```
<v:background xmlns:v="urn:schemas-microsoft-com:vml" fill="t">
<v:fill type="tile" src="<a href="https://i.imgur.com/YJOX1PC.png" rel="nofollow">https://i.imgur.com/YJOX1PC.png</a>" color="#7bceeb"/>
</v:background>
```

* 现在，任何内容都可以插入有效的HTML注释中（下面所示的某些语法之前已被去除）：

`<!--[if gte mso 9]> <![endif]-->`

* 不要用填充空表单元格 `&nbsp;`

* “最大化/最小化”按钮已添加到HTML源代码编辑器中
* 预先存在的表属性现在在“表属性”对话框中标识和显示
* 现在，默认情况下将显示两行按钮。
* 编辑器现在将接受任何元素（甚至已弃用或非标准元素）：

`<myCustomElement>Hello World!</myCustomElement>`

* 编辑器现在将接受任何属性（甚至包括已弃用或非标准属性）：

```
<myCustomElement myCustomAttribute="foo">Hello World!</myCustomElement>
<td background="someImage.png"> 
```
